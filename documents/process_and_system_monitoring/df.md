# Посібник з утиліти `df`

`df` (disk free) — це стандартна утиліта командного рядка, яка використовується для відображення звіту про використання дискового простору файловими системами.

### **Ключова концепція**

На відміну від `du`, яка аналізує простір, зайнятий конкретними директоріями, `df` показує загальну картину на рівні цілих файлових систем (розділів диска, мережевих дисків тощо). Вона не перевіряє окремі файли, а просто запитує у ядра системи: "Скільки всього місця, скільки використано і скільки вільно на цьому диску?".

`df` — це ваш основний інструмент, щоб швидко перевірити, чи не закінчується місце на ваших дисках.

#### **Базовий синтаксис**

```bash
df [опції] [шлях_до_файлу_або_директорії]
```

Якщо вказати шлях, `df` покаже інформацію про ту файлову систему, на якій знаходиться цей шлях.

---

### **Основні опції**

| Опція | Пояснення |
| :--- | :--- |
| **`-h`** | **(Human-readable)**. **Найкорисніша опція.** Показувати розміри у людино-читабельному форматі (Кілобайти, Мегабайти, Гігабайти). |
| **`-H`** | **(SI)**. Схожа на `-h`, але використовує степені 1000 замість 1024 (як у маркетингових матеріалах). Зазвичай `-h` є більш звичною. |
| **`-a`** | **(All)**. Показати всі файлові системи, включно з віртуальними, псевдо- та недоступними. |
| **`-T`** | **(Print type)**. Додати колонку з типом файлової системи (наприклад, `ext4`, `xfs`, `nfs`, `tmpfs`). |
| **`-i`** | **(Inodes)**. Показати інформацію про використання inode, а не блоків. Це важливо для діагностики помилок "диск повний", коли місце наче є, але закінчились inode. |
| **`--total`** | В кінці виводу додати рядок із загальними підсумками. |

---

### **Розуміння виводу команди**

Розглянемо приклад виводу `df -h`:

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        20G   15G  4.0G  80% /
/dev/sdb1       100G   10G   90G  10% /data
tmpfs           3.9G     0  3.9G   0% /dev/shm
```

**Стовпці:**
*   **`Filesystem`**: Назва пристрою або розділу файлової системи.
*   **`Size`**: Загальний розмір файлової системи.
*   **`Used`**: Використаний простір.
*   **`Avail`**: Доступний простір (зазвичай для звичайних користувачів, оскільки частина місця резервується для `root`).
*   **`Use%`**: Відсоток використаного простору.
*   **`Mounted on`**: Точка монтування — директорія, до якої "прикріплена" ця файлова система.

### **Практичні приклади**

1.  **Показати використання диска у зручному форматі:**
    ```bash
    df -h
    ```

2.  **Показати використання диска разом з типами файлових систем:**
    ```bash
    df -hT
    ```

3.  **Перевірити, скільки місця на розділі, де знаходиться ваша домашня директорія:**
    ```bash
    df -h /home
    ```

4.  **Перевірити використання inode:**
    *   Це корисно, якщо ви не можете створити новий файл, хоча `df -h` показує вільне місце. Можливо, у вас закінчились inode.
    ```bash
    df -i
    ```

### **`df` чи `du`? — Підсумок**

Це дві найважливіші команди для аналізу дискового простору, і вони відповідають на різні питання.

*   **`df` (disk free)**: "Скільки вільного місця на всьому диску/розділі?"
    *   *Аналогія: Скільки пального залишилося в баку автомобіля?*

*   **`du` (disk usage)**: "Скільки місця займає ось ця конкретна директорія?"
    *   *Аналогія: Скільки важить валіза у багажнику?*

Використовуйте `df`, щоб отримати загальну картину стану ваших дисків. Використовуйте `du`, щоб знайти, які саме директорії займають найбільше місця.
